DROPEDGE: TOWARDS DEEP GRAPH CONVOLUTIONAL NETWORKS ON NODE CLASSIFICATION

  2020-11-11   note papers reading   papers GNN Node Classification 字数统计: 643字 阅读时长: 2分

DROPEDGE: TOWARDS DEEP GRAPH CONVOLUTIONAL NETWORKS ON NODE CLASSIFICATION

1. 背景

过拟合和过度平滑是当前节点分类的两个主要困难。

2. 解决方法

DropEdge策略： 并不是网络结构的创新，只是在每个epoch后随机丢掉某个数量的边。 处理overfitting： DropEdge像一种数据增强的方法，每次dropedge后相当于得到了原图的变形副本，从邻居信息聚合的角度看，以概率$p$丢弃边后再对边权重归一化，并没有改变邻居聚合的期望，所以这是一种无偏数据增强方法； 处理oversmoothing： DropEdge让节点的连接更稀疏，所以在某种程度上可以避免oversmoothing（过度平滑正是由于网络层数加深，节点汇聚的信息范围越来越大，导致最终各个节点的特征趋于一致）；

具体方法：

假设$E$是边的总数量，$A$为邻接矩阵，$p$是丢弃率，$E_p$是要丢弃的边的数量，则 $$A_{drop} = A - A'$$ 其中$A'$由最初的边$E$的随机子集$E_p$得到，$A_{drop}$重新归一化后为$\hat A_{drop}$。 验证和测试的时候不使用DropEdge。

3. 实验

从实验结果可以看出两点：

  1. 并不是网络越深效果越好，网络的深度要考虑数据集的大小；
  2. 在有的数据集上使用DropEdge策略提升的多，有的却几乎没变化；
  3. IncepGCN效果相对于其他的网络性能略胜一筹；

由此引发思考：

  1. 既然文中说自己的方法能处理过拟合，那就算网络深了，效果最起码不应该变差啊（像ResNet中说的那样），可是这里怎么会越来越差呢？发生了梯度消失？
